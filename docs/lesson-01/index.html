02 — Подходы

Rule-Based vs Machine Learning

Два фундаментально разных подхода к решению задач.

Традиционное программирование

Rule-Based алгоритмы

Использует подход ЕСЛИ X, ТО Y. Человек вручную пишет правила.

  • ✅ Предсказуемость
  • ✅ Прозрачность логики
  • ❌ Нет адаптивности
  • ❌ Сложность масштабирования
  • ❌ Ограничения в сложных задачах
VS

Искусственный интеллект

Machine Learning

Анализирует данные и сам находит закономерности. Принимает решения на основе изученного.

  • ✅ Адаптивность
  • ✅ Масштабируемость
  • ✅ Работает со сложными задачами
  • ❌ Нужны данные для обучения
  • ❌ "Чёрный ящик"

03 — Обучение

Что значит обучить модель?

Датасет — это набор данных, собранных и структурированных для обучения или проверки модели AI.

1
Датасет
Набор данных, размеченных человеком (спам / не спам)
2
Обучение
Модель тренируется на размеченных данных
3
Валидация
Проверка на новых, но тоже размеченных данных
4
Тестирование
Финальная проверка на данных, которых модель не видела

Метрики качества модели

🎯
Accuracy

Общая правильность

правильные / все
Из 100 писем модель правильно определила 90 → accuracy = 90%
🔍
Precision

Насколько можно доверять

верный спам / весь отмеченный спам
Модель отметила 20 как спам, из них 15 реально спам → precision = 75%
📡
Recall

Насколько не пропустила

найденный спам / весь реальный спам
Из 30 реальных спам-писем модель нашла 15 → recall = 50%

06 — Нейросети

Нейронные сети

Метод в ML, вдохновлённый работой мозга. Нейросеть — это цепочка математических операций, где данные проходят через слои «нейронов», каждый из которых выделяет всё более сложные закономерности.

💡 Аналогия

Представьте команду экспертов, работающих по цепочке. Первый замечает простые признаки («есть текст»), второй — более сложные («текст про скидку»), третий выносит решение («это спам»). Каждый «нейрон» — такой эксперт, а «вес» — степень его влияния на итоговое решение.

Как данные проходят через нейросеть

x1
Входной слой
Сюда попадают сырые данные — то, что мы подаём нейросети. Каждое число — это отдельный нейрон.
Пример: фото яблока
R=255   G=50   B=0   size=0.7
R255
G50
B0
size0.7
h1
Скрытый слой 1
Ищет простые признаки: есть ли край, какой цвет, есть ли тень. Каждый нейрон умножает входы на веса и суммирует.
Что видят нейроны:
край? 0.82   цвет? 0.95   тень? 0.33
край?0.82
цвет?0.95
линия?0.41
тень?0.33
h4
Скрытый слой 2
Комбинирует простые признаки в сложные понятия: «круглое», «красное», «гладкое». Чем глубже слой — тем абстрактнее признак.
Что видят нейроны:
круглое 0.91   красное 0.88   гладкое 0.67
круглое0.91
красное0.88
гладкое0.67
y
Выходной слой
Финальный ответ — вероятность каждого варианта. Нейросеть не говорит «точно яблоко», а даёт проценты уверенности.
Результат:
🍎 яблоко: 95%   🍅 5%
🍎95%
🍅5%

Пример: нейросеть распознаёт яблоко на фото. Каждый слой выделяет всё более сложные признаки — от пикселей до формы и цвета.

⚖️ Что такое вес?

Вес — число (например, 0.7), которое определяет, насколько сильно один нейрон влияет на следующий. При обучении модель подбирает миллиарды таких чисел, чтобы давать правильные ответы.

📚 Что такое Deep Learning?

Deep Learning = нейросеть с множеством скрытых слоёв (десятки, сотни). GPT-4 имеет ~120 слоёв и ~1.8 трлн параметров (весов). Больше слоёв = более сложные закономерности.

07 — Трансформер

Как работает GPT

GPT = Generative Pretrained Transformer. Архитектура нейронной сети, разработанная в 2017 году, ставшая основой современного AI.

1

Токенизация

Разбиение входного текста на небольшие фрагменты — токены. Токен может быть целым словом, частью слова или знаком.

Привет , как дела ?
2

Присвоение ID

Каждому токену присваивается уникальный числовой ID на основе словаря модели.

Привет → 15339    , → 11    как → 2482    дела → 9017    ? → 30
3

Эмбеддинги

ID преобразуется в вектор из сотен чисел, кодирующий смысл токена. Близкие по смыслу слова имеют близкие векторы.

"кот" → [0.12, -0.45, 0.78, ...] (768 чисел)
"котёнок" → [0.14, -0.42, 0.75, ...] — близко к "кот"
"машина" → [0.91, 0.33, -0.12, ...] — далеко от "кот"
4

Механизм внимания (Attention)

Каждое слово "смотрит" на все остальные слова, определяя их важность для контекста. Слово "банк" получает разные эмбеддинги в "речной банк" и "банк данных".

Энкодер: понимает входной текст → Декодер: генерирует ответ
5

Генерация текста

Модель вычисляет вероятности следующего токена и выбирает наиболее подходящий. Процесс повторяется для каждого нового токена.

"Напиши рецепт яичницы" → $ → "Чтобы" → "приготовить" → "яичницу" → "нужно" → "разбить" → "яйца" ...
