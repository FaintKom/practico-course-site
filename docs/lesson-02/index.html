💡 Что такое промпт?

Промпт — это входные данные, которые вы передаёте модели для получения желаемого результата.

Качество промпта напрямую определяет качество ответа. Хороший промпт — это чёткий, контекстуальный и структурированный запрос.

⚠️ 3 распространённые ошибки

🌫️
Размытая формулировка
Нечёткие запросы без конкретных деталей. Модель не понимает, что именно от неё нужно.
Пример: «Расскажи про маркетинг»
📦
Перегрузка информацией
Слишком много несвязанных деталей сбивают модель и ухудшают результат.
Пример: «Напиши про маркетинг, финансы, IT и HR сразу»
🚫
Нет примеров и контекста
Без примеров и контекста модель вынуждена угадывать формат и стиль ответа.
Пример: «Сделай таблицу» (без данных и формата)
02 — Классификация

Типы промптов

Теперь, когда мы знаем основы, разберём какие бывают промпты — по функции, контексту и области применения.

По функции

⚙️
Системные промпты
Настраивают AI на определённую роль и поведение. Задают контекст для всего диалога.
«Ты — опытный HR-менеджер. Отвечай кратко, профессионально.»
🔀
Условные промпты
Содержат логику «если/то». Позволяют модели адаптировать ответ в зависимости от условий.
«Если вопрос про возврат — предложи форму. Если про доставку — дай трекинг.»
💬
Промпты на ест. языке
Обычные запросы в свободной форме. Самый простой тип, используемый в повседневном общении с AI.
«Помоги мне составить план презентации по продажам.»

По контексту и стилю

🎯
Эксплицитные (Explicit)
Высокая специфичность. Чёткое описание задачи, формата и ограничений. Минимум пространства для интерпретации.
🌀
Имплицитные (Implicit)
Оставляют пространство для интерпретации. Модель сама решает, как лучше подойти к задаче.
🎨
Креативные (Creative)
Направлены на генерацию оригинальных идей. Поощряют нестандартное мышление и эксперименты.

По области применения

📚
Информационные
Поиск фактов и данных
💬 «Расскажи о текущих трендах AI в ритейле за 2025 год»
✏️
Креативные
Генерация идей и контента
💬 «Придумай концепцию рекламной кампании для экологичной упаковки»
📋
Инструктивные
Пошаговые инструкции
💬 «Объясни пошагово, как настроить CRM для отдела продаж из 10 человек»
❓
Уточняющие
Разъяснение концепций
💬 «Что именно означает 'product-market fit' в контексте B2B SaaS?»
⚖️
Сравнительные
Анализ альтернатив
💬 «Сравни преимущества и недостатки Slack vs Microsoft Teams для команды из 50 человек»
🔮
Гипотетические
«Что если» сценарии
💬 «Что произойдёт, если компания полностью автоматизирует клиентскую поддержку через AI?»
📊
Аналитические

03 — Структура

Элементы хорошего промпта

Четыре ключевых компонента, из которых строится эффективный промпт.

1
Инструкция
Instructions

Что именно модель должна сделать. Чёткое действие или задача.

❌ Плохо
«Что-нибудь про маркетинг»
✅ Хорошо
«Составь план контент-маркетинга на Q1»
2
Контекст
Context

Фоновая информация, которая помогает модели понять ситуацию.

❌ Плохо
«Напиши письмо»
✅ Хорошо
«Напиши письмо клиенту B2B SaaS, который не продлил подписку»
3
Входные данные
Input Data

Конкретные данные для обработки: текст, числа, примеры.

❌ Плохо
«Проанализируй продажи»
✅ Хорошо
«Проанализируй: Q1=$120K, Q2=$95K, Q3=$140K»
4
Индикатор вывода
Output Indicator

Желаемый формат, стиль, длина и структура ответа.

❌ Плохо
«Дай ответ»
✅ Хорошо
«Ответ в виде таблицы: столбцы Метрика, Q1, Q2, Изменение %»

Формула хорошего промпта

Инструкция + Контекст + Данные + Формат вывода = ✨ Отличный результат

06 — Контекст

Контекст и контекстное окно

Мы упоминали контекст как элемент промпта — теперь разберём, как модель его обрабатывает и почему это важно.

💬
История диалога
Все предыдущие сообщения в текущем чате. Модель учитывает их при генерации ответа.
⚙️
Системный промпт
Скрытые инструкции, которые задают роль и поведение модели (Custom Instructions, system message).
📂
Загруженные данные
Файлы, документы, изображения, которые вы прикрепляете к диалогу (RAG, вложения).

📦 Контекстное окно (Context Window)

Максимальный объём текста, который модель может «видеть» одновременно. Измеряется в токенах.

Что такое токен?

Токен — это фрагмент текста (слово, часть слова или символ). В среднем:

  • 🇬🇧 1 токен ≈ 0.75 слова (английский)
  • 🇷🇺 1 токен ≈ 0.5 слова (русский)
  • 📖 1000 токенов ≈ 600 слов на русском
Размеры окна (2025-2026)
  • GPT-4o (128K) / GPT-5 (1M)
  • Claude Opus 4 (1M)
  • Gemini 2.5 Pro (1M)
  • LLaMA 4 (128K)

Почему контекст важен для промптинга?

🎯
Точность ответов
Больше контекста = более релевантный и точный ответ
💰
Экономия токенов
Управление контекстом снижает стоимость API-запросов
⚡
Скорость работы
Меньше итераций = быстрее достигается нужный результат

✅ Когда использовать роли

  • 📢 Простой фактический вопрос

  • 🔢 Арифметические или логические задачи
  • 🔍 Поиск информации без аналитики
08 — Детали

TAG, RISE, BAB, RTF

Подробный разбор каждого фреймворка с примерами промптов.

TAG — Task, Action, Goal

Простой и быстрый. Идеален для рутинных задач.
T ask — Задача
A ction — Действие
G oal — Цель
Пример:
Task: Написать email клиенту
Action: Составь вежливое письмо с напоминанием об оплате счёта
Goal: Клиент оплачивает счёт в течение 3 дней

RISE — Role, Input, Steps, Expectation

Для задач, где важна последовательность шагов и экспертная роль.
R ole — Роль
I nput — Вход
S teps — Шаги
E xpectation — Ожидание
Пример:
Role: Ты — бизнес-аналитик с опытом в e-commerce
Input: Данные о продажах за Q1-Q3 и отзывы клиентов
Steps: 1) Проанализируй тренды продаж 2) Выдели проблемные категории 3) Сопоставь с отзывами
Expectation: Отчёт с 5 ключевыми выводами и рекомендациями

BAB — Before, After, Bridge

Фреймворк трансформации. Опишите текущее состояние, желаемое и путь между ними.
B efore — До
A fter — После
B ridge — Мост
Пример:
Before: Наш onboarding занимает 2 недели, 40% новичков уходят в первый месяц
After: Onboarding за 5 дней, удержание 90% сотрудников
Bridge: Предложи план трансформации onboarding-процесса с конкретными шагами и таймингами

RTF — Role, Task, Format

Универсальный фреймворк для генерации контента с заданным форматом.
R ole — Роль
T ask — Задача
F ormat — Формат
Пример:
Role: Ты — SMM-менеджер для IT-компании
Task: Напиши серию из 5 постов для LinkedIn о запуске нового продукта
Format: Каждый пост: заголовок (до 10 слов), тело (до 150 слов), 3 хештега, CTA

09 — AUTOMAT

Фреймворк AUTOMAT

Самый детальный фреймворк из 7 компонентов. Для сложных, многослойных задач.

A
Act — Роль
Определите, кем должна быть модель. Задайте экспертизу и опыт.
«Ты — маркетинг-директор с 10-летним опытом в SaaS»
U
User — Аудитория
Для кого предназначен результат? Уровень знаний аудитории.
«Для CEO стартапов без технического бэкграунда»
T
Target — Трансформация
Какое изменение должен произвести результат? Из точки A в точку B.
«Превратить хаотичный маркетинг в системную стратегию»
O
Output — Формат вывода
В каком формате нужен ответ: таблица, список, отчёт, код.
«Таблица с колонками: Канал, Бюджет, KPI, Срок»
M
Model — Модель
Укажите пример или эталон, на который нужно ориентироваться.
«По модели McKinsey 7S framework»
A
Additional — Ограничения
Дополнительные условия: длина, язык, что исключить.
«Не более 2 страниц. Без технического жаргона. На русском языке.»
T
Tone — Тон
Стиль коммуникации: формальный, дружественный, академический.
«Профессиональный, но доступный. Без канцеляризмов.»

📝 Полный пример AUTOMAT

A (Act): Ты — маркетинг-директор с 10-летним опытом в SaaS
U (User): Для CEO стартапов без технического бэкграунда
T (Target): Создать стратегию выхода на новый рынок
O (Output): Таблица с колонками: Канал, Бюджет, KPI, Срок
M (Model): По модели Go-to-Market Strategy
A (Additional): Бюджет до $50K, фокус на B2B, рынок EU
T (Tone): Профессиональный, с конкретными цифрами
10 — Техники

Техники промптинга

Фреймворки помогают структурировать промпт. Теперь изучим техники, которые меняют сам способ рассуждения модели.

1
Zero-shot
Без примеров
Прямая инструкция без примеров. Модель опирается только на свои знания. Подходит для простых задач.
«Классифицируй отзыв как позитивный, негативный или нейтральный: "Товар пришёл вовремя, но упаковка помята"»
2
Few-shot
Несколько примеров
Даёте 2-5 примеров перед задачей. Модель учится формату из примеров. Важна консистентность формата.
Отзыв: "Отличное качество!" → Позитивный
Отзыв: "Не рекомендую" → Негативный
Отзыв: "Нормально, ничего особенного" → ?
3
Chain-of-Thought (CoT)
Цепочка рассуждений
Просите модель рассуждать шаг за шагом. Значительно повышает точность в сложных задачах.
«Давай решим эту задачу шаг за шагом. Сначала определим ключевые переменные, затем выстроим логику...»
4
Tree of Thoughts (ToT)
Дерево мыслей
Модель рассматривает несколько путей рассуждения параллельно и выбирает лучший. Для сложных решений.
«Рассмотри 3 варианта решения этой проблемы. Для каждого: опиши плюсы, минусы, риски. Выбери лучший.»
5
Self-Criticism
Самокритика
Цикл: генерация → критика → улучшение. Модель сама находит и исправляет ошибки в своём ответе.
«Сгенерируй ответ, затем найди в нём 3 слабых места и перепиши с учётом критики.»
6
Step-back Prompting
Шаг назад
Сначала задаёте общий вопрос для контекста, затем конкретный. Помогает модели увидеть полную картину.
Шаг 1: «Какие основные принципы ценообразования в SaaS?»
Шаг 2: «Исходя из этого, как нам оценить наш продукт для рынка SMB?»
7
Re-Reading (Re2)
Перечитывание
Повторяете ключевую часть запроса для акцента. Помогает модели сфокусироваться на важном.
«Проанализируй данные о продажах. Обрати особое внимание на сезонность. Повторяю: ключевой фокус — сезонные паттерны.»
8
Rephrase and Respond (RaR)
Перефразируй и ответь
Просите модель сначала перефразировать вопрос своими словами, потом ответить. Улучшает понимание задачи.
«Сначала перефразируй мой вопрос своими словами, чтобы убедиться что ты правильно понял, затем дай ответ.»

12 — Настройка

Настройка ChatGPT

Приватность, Custom Instructions и создание собственных GPT.

🔒 Настройки приватности

По умолчанию ChatGPT может использовать ваши данные для обучения. Важно настроить приватность:

1. Отключить обучение

Settings → Data Controls → Improve the model → Off

2. Временные чаты

Используйте Temporary Chat для конфиденциальных данных — они не сохраняются

3. Team/Enterprise

Для бизнеса: данные не используются для обучения по умолчанию

Создание CustomGPT

Список ваших GPT — My GPTs

1

Определите задачу

Опишите, для чего нужен ваш GPT: роль, аудитория, типичные запросы.

2

Напишите Instructions

Системный промпт, определяющий поведение. Используйте фреймворк AUTOMAT.

3

Загрузите знания

Добавьте файлы (PDF, DOCX, CSV) — GPT будет использовать их как базу знаний (RAG).

4

Настройте и протестируйте

Выберите capabilities (Web, DALL-E, Code), протестируйте с типичными запросами, итерируйте.

🎯 Практика

Практическое задание

Примените изученные фреймворки и техники на реальной задаче.

📋 Задание

Напишите промпт для генерации контент-стратегии для социальных сетей компании по производству продуктов питания. Используйте один из изученных фреймворков (TAG, RISE, BAB, RTF или AUTOMAT).

✅ Хороший пример (RISE)
Role: Ты — SMM-стратег с 7-летним опытом в FMCG.

Input: Компания «ЭкоФерма» — органические продукты, ЦА: женщины 28-45, ценности: здоровье, экология, семья. Каналы: Instagram, Telegram. Бюджет: 150K руб/мес.

Steps:
1. Проанализируй ЦА и выдели 5 ключевых болей
2. Предложи контент-план на месяц (20 постов)
3. Для каждого поста укажи формат и тему
4. Добавь 3 идеи для Reels/Stories

Expectation: Таблица с контент-планом + отдельный блок с идеями для видео.
❌ Плохой пример
Напиши контент-план для продуктовой компании в соцсетях.

❌ Нет роли — модель не знает с какой экспертизой отвечать

❌ Нет контекста — какая компания, какая аудитория?

❌ Нет структуры — что именно нужно в контент-плане?

❌ Нет формата — таблица? текст? на какой период?

💡 Подсказка

Попробуйте написать промпт в ChatGPT прямо сейчас. Начните с плохого варианта, затем улучшите его с помощью фреймворка. Сравните результаты — разница будет очевидна!
